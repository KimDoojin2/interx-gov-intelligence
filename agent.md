# AGENT.md

## 프로젝트 목적
이 프로젝트는 정부지원사업 공고를 수집, 정규화, 점수화하고,
InterX BD CRM 시트 구조에 맞춰 업로드하는 엔진이다.

## 절대 원칙
1. 거대한 단일 파일을 만들지 않는다.
2. 새 기능은 반드시 적절한 레이어에 배치한다.
3. 도메인 규칙(core)은 외부 의존성을 import하지 않는다.
4. application은 use case 중심으로 작성한다.
5. infrastructure는 외부 입출력 구현만 담당한다.
6. interfaces는 실행 진입점만 둔다.
7. 컬럼명/시트명/점수규칙은 코드에 하드코딩하지 말고 config로 분리한다.
8. 함수가 50줄을 넘기면 분리 가능성을 먼저 검토한다.
9. 파일이 300줄을 넘기면 역할 분리를 검토한다.
10. notebook은 실험용이며 운영 코어 로직은 src 아래에 둔다.

## 금지 사항
- crawler 안에 비즈니스 규칙 작성 금지
- util.py 하나에 모든 공용 함수 몰아넣기 금지
- pandas DataFrame 안에서 핵심 도메인 규칙 처리 금지
- print 디버깅 남발 금지
- try/except로 에러를 삼키지 말 것

## 리팩토링 트리거
- 같은 로직 2회 이상 반복
- site별 if/elif 분기 증가
- 함수 50줄 이상
- 파일 300줄 이상
- 시트 컬럼 변경 시 여러 파일을 동시에 수정해야 하는 구조

## 작업 순서
1. 요구사항을 use case로 해석
2. 영향받는 레이어 식별
3. 기존 포트/정책/엔티티 재사용 검토
4. 새 코드 추가
5. 테스트 또는 샘플 검증
6. 문서 업데이트
